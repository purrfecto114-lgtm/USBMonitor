# USB Monitor 1.0 实施说明

## 本轮目标

将产品评审中的高优先级问题直接落地到代码，重点处理安全弹出、自动化配置、持久反馈和最近记录四个闭环。

## 已实施

1. **安全弹出状态机**
   - 所有任务统一使用 `E:` 形式的盘符键。
   - 同一盘符任务执行中拒绝重复请求。
   - `QThread` 由盘符强引用保存，直到 Qt 的 `finished` 信号触发后再清理。
   - 成功后 350ms 触发一次协调器扫描，减少菜单中残留的旧设备项。

2. **自动化配置闭环**
   - `ConfigStore.load/save` 已读写 `hooks`。
   - 规则数量、名称、命令参数和防抖时间均有边界校验。
   - 只允许 `{path}`、`{label}` 两个占位符。
   - 子进程显式使用 `shell=False`，输出重定向到 `DEVNULL`，避免高输出命令占满内存。

3. **可靠反馈**
   - 托盘右键菜单顶部保留最近任务状态。
   - 托盘 tooltip 同步显示设备数和最近操作结果。
   - 设备正在弹出时，对应菜单项变为“正在安全弹出…”并禁用。

4. **最近记录可用化**
   - 左键设备菜单新增“最近使用”。
   - 在线记录可直接打开和定位；离线记录显示最后使用时间。
   - 每条记录支持复制盘符，并保留统一清空入口。
